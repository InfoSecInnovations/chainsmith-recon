Phase 19 — Distribution and Packaging
======================================

Future work to make Chainsmith installable and runnable via standard
package managers and container registries.

Status: NOT YET IMPLEMENTED
These items were previously listed in the README as if they existed.
They are aspirational and should only be documented as available once
they are actually working.


1. PyPI Distribution
--------------------
Publish to PyPI so users can install with:

    pip install chainsmith-recon

Requires:
  - Finalize pyproject.toml metadata (version is currently 1.2.0 in
    pyproject.toml vs 1.3.0 in cli.py — reconcile)
  - Set up CI/CD pipeline for automated PyPI publishing
  - Create a console_scripts entry point so `chainsmith` works after
    pip install without manual PATH setup
  - Decide on namespace: chainsmith-recon or just chainsmith
  - Test installation in clean virtualenvs on Linux, macOS, Windows


2. Docker Image
---------------
Publish an official Docker image:

    docker pull <registry>/chainsmith-recon:latest
    docker run -it <registry>/chainsmith-recon scan example.com

Requires:
  - Finalize Dockerfile (docker-compose.chainsmith.yml exists but no
    standalone Dockerfile for distribution)
  - Choose a registry (Docker Hub, GHCR, etc.)
  - Set up automated image builds on release
  - Decide on image variants (slim, full with dev tools, etc.)
  - Handle persistent data (SQLite DB) via volume mounts


3. GitHub Repository
--------------------
The README previously referenced:

    https://github.com/infosecinnovations/chainsmith-recon

This URL does not exist yet. Decide on:
  - Organization name and repo location
  - Public vs private
  - Issue templates, CI workflows, release automation


4. Install-from-Source Instructions
-----------------------------------
Once the repo is public, the README should include:

    git clone https://github.com/<org>/chainsmith-recon.git
    cd chainsmith-recon
    pip install -e .

Currently the README just says `git clone <repo-url>` as a placeholder.
